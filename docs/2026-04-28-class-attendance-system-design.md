## 1. 项目概述

### 1.1 项目背景

本课程设计面向教学场景中的班级考勤需求，采用 B/S（Browser/Server）架构实现一个融合“基础考勤、合照识别、情绪分析”的班级考勤系统。系统通过浏览器采集图像，由后端完成人脸识别、记录考勤结果、统计活动参与情况，并展示情绪分析结果。

考虑到课程设计的时间与实现复杂度，本项目优先完成一个“基础功能完整、流程可演示、代码结构清晰”的最小可用版本。在此基础上，预留活体检测、权限控制、抗攻击增强等接口，便于后续扩展与答辩说明。

### 1.2 项目目标

本阶段系统目标如下：

1. 实现基础考勤流程：摄像头采集或上传图像，后端识别学生身份并生成考勤记录。
2. 实现合照批量识别：上传班级活动照片，识别照片中的学生并生成参与名单。
3. 实现情绪分析展示：在考勤和合照识别中同步返回情绪分类结果，并支持统计查看。
4. 实现数据管理与导出：支持学生信息维护、考勤记录查询、Excel 导出、活动参与统计。
5. 预留安全扩展能力：保留活体检测、权限控制等接口，但本阶段不做强实现。

## 3. 系统设计

### 3.1 总体架构

系统采用三层 B/S 架构：

```text
前端层（Vue 3）
  ├─ 登录页
  ├─ 学生管理页
  ├─ 考勤页
  ├─ 合照识别页
  └─ 统计页

后端层（FastAPI）
  ├─ 认证模块
  ├─ 学生管理模块
  ├─ 考勤模块
  ├─ 合照识别模块
  ├─ 情绪统计模块
  └─ 安全扩展接口

数据层（SQLite）
  ├─ user
  ├─ student
  ├─ face_feature
  ├─ attendance
  ├─ activity
  └─ activity_participant
```

### 3.2 本阶段技术选型

为保证系统能够快速落地，本阶段采用“可运行优先”的技术路线：

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Element Plus + ECharts |
| 后端 | FastAPI + SQLAlchemy + Pydantic |
| 数据库 | SQLite |
| 数据导出 | openpyxl |
| 图像处理 | 统一封装于服务层，先采用可替换的基础实现 |
| 情绪识别 | 简化情绪分析服务，后续可替换为 DeepFace |
| 活体检测 | 预留服务接口，后续可接入 MediaPipe / CNN 模型 |

### 3.3 识别流程设计

#### 3.3.1 基础考勤流程

```text
前端打开摄像头
→ 用户手动拍照或上传图像
→ 前端提交图片到 /attendance/check
→ 后端进行基础人脸识别
→ 返回学生信息、考勤状态、时间、情绪结果
→ 写入 attendance 表
```

#### 3.3.2 合照识别流程

```text
前端上传合照
→ 后端创建活动记录
→ 后端识别合照中的学生
→ 生成参与名单
→ 写入 activity 和 activity_participant 表
→ 返回识别结果与统计信息
```

#### 3.3.3 情绪分析流程

```text
图像提交到后端
→ 情绪分析服务返回情绪标签
→ 与考勤记录或活动参与记录一并存储
→ 前端统计页调用接口查看情绪分布
```

### 3.5 接口设计

#### 3.5.1 通用规范

- 基础路径：`/api/v1`
- 数据格式：JSON / multipart/form-data
- 响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

#### 3.5.2 本阶段实现接口

**认证模块**

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /auth/login | 用户登录 |
| GET | /auth/me | 获取当前用户信息 |
| POST | /auth/logout | 用户登出 |

**学生管理模块**

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /students | 获取学生列表 |
| POST | /students | 新增学生 |
| PUT | /students/{id} | 更新学生 |
| DELETE | /students/{id} | 删除学生 |
| POST | /students/{id}/face | 上传或更新学生人脸照片 |
| POST | /students/batch | 批量导入学生基础信息 |

**考勤模块**

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /attendance/check | 提交单张图片进行考勤 |
| GET | /attendance/records | 查询考勤记录 |
| GET | /attendance/export | 导出考勤记录 Excel |
| GET | /attendance/action-challenge | 活体挑战接口占位 |

**合照识别模块**

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /group/upload | 上传合照并识别 |
| GET | /group/activities | 查询活动列表 |
| GET | /group/activities/{id} | 查询活动详情 |
| GET | /group/statistics | 查询活动参与统计 |

**情绪统计模块**

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /emotion/statistics | 获取情绪统计 |
| GET | /emotion/timeline | 获取情绪时间线 |

#### 3.5.3 预留接口说明

以下接口在本阶段仅提供占位返回，不做完整算法实现：

| 接口 | 预留目的 |
|------|----------|
| /attendance/action-challenge | 后续接入动作活体检测 |
| 权限依赖 require_teacher / require_student | 后续接入角色权限控制 |
| liveness_service | 后续接入照片/视频攻击检测 |

## 4. 本阶段实现边界说明

### 4.1 已纳入本阶段实现

1. 后端项目骨架与数据库建表
2. 登录认证与基础用户信息接口
3. 学生管理与人脸图片上传
4. 基础考勤提交、记录查询、导出
5. 合照识别、活动记录、参与统计
6. 情绪统计接口
7. 异常处理与统一返回结构

### 4.2 暂不做强实现但保留设计

1. 动作活体检测
2. 纹理活体检测
3. 视频攻击防御
4. 教师/学生细粒度权限控制
5. 高精度深度学习模型部署

### 4.3 后续扩展方向

1. 将占位识别服务替换为真实人脸识别模型
2. 将简化情绪服务替换为 DeepFace 等模型
3. 接入 MediaPipe 动作活体和后端活体二次校验
4. 完成教师端与学生端的权限分离
5. 增加日志审计与系统监控
