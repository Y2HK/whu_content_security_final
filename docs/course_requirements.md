**《内容安全实验》课程设计及评分标准**

**一、课程设计总体目标**

本课程设计以内容安全领域的人脸识别、活体检测技术为核心，结合BS（Browser/Server）架构开发，实现一套功能完整、安全可靠的班级考勤系统。要求融入合照学生识别、学生面部情绪分析功能，培养学生对计算机视觉技术、前后端交互、数据安全处理的实践能力，理解人脸识别技术在内容安全场景中的应用边界与防护要点，完成从需求分析到系统部署的全流程开发与测试，提升工程实践与问题解决能力。

**二、课程设计核心任务**

基于BS架构，开发一套融合"基础考勤+合照识别+情绪分析"的班级考勤系统，具体完成以下核心任务：

1.  基础考勤功能：前端通过浏览器调用摄像头采集学生人脸信息，传输至后端；后端完成活体检测、人脸库比对，将考勤结果（成功/失败、匹配学生信息、考勤时间）返回至前端渲染显示，实现日常班级考勤的自动化记录。

2.  合照学生识别功能：支持上传班级/活动大合照，后端对照片中的人脸进行批量识别，匹配人脸库中学生信息，生成匹配成功的学生名单，统计学生参与各类活动的频次，形成统计报表。

3.  情绪分析功能：在考勤、合照识别过程中，同步对学生面部表情进行分析，并对情绪进行分类，可在前端查看统计结果。

4.  系统安全性能：优化识别算法，提升活体检测、人脸比对、合照识别的准确率，降低误识别率；实现系统异常处理（如摄像头调用失败、照片上传失败、识别超时等）。

5.  文档与测试：完成课程设计报告，包含需求分析、系统设计、代码实现、结果分析等内容，提交可运行的系统源码及相关文档。

**三、课程设计具体要求**

**（一）架构要求（10分）**

1.  严格遵循BS架构设计，明确前端、后端、数据库的分层架构，前端负责交互与渲染，后端负责业务逻辑与算法处理，数据库负责数据存储。

2.  前端采用HTML、CSS、JavaScript（或Vue、React等框架）开发，支持主流浏览器（Chrome、Edge等），界面简洁、交互友好。

3.  后端可选用Django、Flask等框架进行开发，提供API接口，实现与前端的数据交互、人脸处理、情绪分析等核心逻辑。

4.  任选数据存储方式（如
    MySQL、SQLite等），需合理设计学生信息、人脸特征、考勤记录、情绪数据、活动参与频次等数据的组织结构。

**（二）核心功能要求（60分）**

**1. 基础考勤功能（25分）**

（1）前端（12分）：实现摄像头调用、人脸实时采集，支持手动拍摄/自动捕捉人脸，显示采集状态；接收后端返回的考勤结果，实时渲染（如显示学生姓名、学号、考勤状态、时间），支持考勤记录查询（按日期、学号筛选）。

（2）后端（13分）：实现活体检测功能（可采用动作活体、纹理活体等方式，防止照片、视频欺骗）；建立班级人脸库；记录考勤数据（学号、姓名、考勤时间、考勤状态、活体检测结果），支持数据导出（如Excel格式）。

**2. 合照学生识别功能（9分）**

（1）人脸识别（5分）：支持前端上传合照，后端对合照进行人脸检测、批量识别，匹配人脸库中的学生信息，生成参与活动的学生名单（包含学号、姓名）。合照识别准确率≥85%，支持处理多人合照（最少10人）。

（2）数据统计（4分）：实现活动频次统计功能，自动记录每位学生参与活动的次数，形成统计报表（如统计数字、表格、柱状图、折线图）。

**3. 情绪分析功能（8分）**

（1）情绪识别（5分）：在考勤人脸采集、合照识别过程中，同步提取学生面部特征，进行情绪分析与分类。

（2）数据统计（3分）：后端记录情绪数据（关联学号、姓名、识别时间、情绪类型），前端支持查看情绪统计。

**4. 系统安全性要求（18分）**

（1）抗攻击能力（15分）：重点强化活体检测的抗伪造能力，能够有效抵御照片、视频伪造攻击，杜绝虚假考勤情况。

（2）权限与异常防护（3分）：实现合理的权限控制，区分教师账号与学生账号权限（教师可查看所有数据，学生仅可查看个人相关记录）。

**（三）实验报告要求（30分）**

（1）课程设计报告（20分）：包含需求分析（功能需求、性能需求、安全需求）、系统设计（架构设计、数据库设计、接口设计、算法设计）、代码实现（核心代码片段、注释说明）、测试用例（针对各功能的测试场景、预期结果、实际结果）、结果分析（识别准确率、系统性能、存在问题及改进方案）、总结与展望等内容，格式规范。

（2）源码提交（10分）：提交完整的系统源码（前端、后端、数据库脚本），代码结构清晰、注释规范，可直接部署运行；附部署说明（明确运行环境、部署步骤、依赖包安装方法）。

**五、注意事项**

1.不得泄露用于系统测试的学生人脸信息、个人隐私，禁止使用未经授权的人脸数据集，不得用于课程设计以外的用途。

2.小组独立完成课程设计，禁止抄袭他人作品，一经发现，课程设计成绩按0分处理，并按学校相关规定严肃处理。

3.按时提交相关材料（课程设计报告、源码等），逾期提交视为缺交，成绩按不合格处理。

4.系统开发过程中，注重代码规范、注释清晰，便于教师检查和后期维护；鼓励创新，可在要求基础上优化功能、提升用户体验。

**六、成绩等级划分**

1.优秀（90-100分）：完全满足所有设计要求，系统功能完整、性能优秀，识别准确率达标，文档规范，有创新点；

2.良好（80-89分）：满足所有设计要求，系统功能完整，识别准确率基本达标，文档规范，无明显bug；

3.中等（70-79分）：基本满足设计要求，核心功能实现，识别准确率基本达标，文档基本规范，存在少量bug；

4.及格（60-69分）：基本满足核心设计要求，主要功能实现，识别准确率偏低，文档不够规范，存在一定bug；

5.不及格（60分以下）：未完成核心功能，系统无法正常运行，文档不完整，或存在抄袭情况。
